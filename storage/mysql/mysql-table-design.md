# MySQL表设计规范

## 第一范式

　　第一范式：列的原子性，即每一列（每一个属性，字段）都不可分割。 举例：销售成本=成本的单价\*销售的数量，所以这里就不可以以销售成作为字段。

## 第二范式

　　第二范式：非主属性必须完全依赖于主属性，不能存在只依赖于主属性的一部分属性。简单而言就是每条数据都需要有唯一主键。

## 第三范式

　　第三范式：消除依赖的传递，任何非主属性不能依赖于其他非主属性。就是非主键之间不能存在依赖关系，必须只能依赖于主键。

## 反范式

反范式是建立在设计数据库表的时候，遵守三大范式的前提，通过建立适当的冗余字段来减少表关联，这些冗余字段需要代码手动进行维护了。

## 索引的选择

1. **选择常用于搜索和过滤的字段:** 选择那些经常出现在查询的 WHERE 子句中的字段。这些字段可能是用于过滤数据的条件，如等值匹配、范围查询等
2. **选择唯一性高的字段：** 如果某个字段的值几乎都是唯一的，那么为这个字段创建唯一索引可以帮助加速检索，也可以用来保证数据的唯一性
3. **注意复合索引的顺序：** 优先选择最常用于过滤的字段作为复合索引的前缀
4. **避免过长的索引字段：** 索引字段越长，索引的大小也越大，这可能会影响性能。尽量选择短而有意义的字段作为索引
